## 1. 产品概述

### 1.1 一句话描述

专为小红书创作者打造的 AI 文案生成工具——输入主题，秒出高质量、有"人味儿"的小红书文案。

### 1.2 目标用户

- 小红书博主（不论粉丝量）
- 想做小红书但不会写文案的新手
- 需要量产内容的小商家/电商运营

### 1.3 核心价值

节省 80% 文案写作时间，产出质量稳定的、不"AI 味"的小红书文案。

### 1.4 为什么现在做

- 小红书月活 3 亿，每天数百万条新笔记
- 专为小红书优化的中文 AI 文案工具极少
- 通用 ChatGPT 写的小红书文案"AI 味"太重
- Gemini 免费 API 中文质量够好，零成本起步

## 2. 产品设计

### 2.1 核心用户流程

```
用户访问 → 输入创作主题（如"推荐一款平价面霜"）
         → 选择内容类型：种草笔记 / 干货分享 / 好物测评 / 生活记录 / Vlog
         → 选择人设风格：可爱 / 知性 / 真实分享 / 搞笑 / 专业
         → 点击「生成文案」
         → 输出：
           ├── 3 个爆款标题候选
           ├── 完整正文（200-800 字）
           ├── 推荐话题标签（5-10 个）
           └── 封面文字建议
         → 一键复制 / 微调重新生成
```

### 2.2 MVP 功能清单

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P0 | 文案生成核心 | 主题 → 标题 + 正文 + 话题 + Emoji |
| P0 | 内容类型选择 | 种草/干货/好物测评/生活记录/Vlog（5 种） |
| P0 | 人设风格定制 | 5 种风格预设，让文案有"人味儿" |
| P0 | 免费额度控制 | 未登录 1 次/天，注册用户 3 次/天 |
| P1 | 用户注册登录 | 邮箱注册 + Google/GitHub OAuth |
| P1 | 会员订阅 | ¥19.9/月 Pro，¥49.9/月 Pro+ |
| P1 | 文案历史记录 | 查看、搜索、删除自己生成过的文案 |
| P2 | 文案收藏 | 收藏满意的文案 |
| P2 | 风格自定义 | 用户填写自己的人设描述 |
| P2 | 文案分享 | 生成分享卡片，方便发到微信群 |

### 2.3 "去 AI 味"策略（核心竞争力）

- Prompt 植入真实小红书博主语言习惯（"姐妹们"、"谁懂啊"、"绝绝子"）
- 控制句子长度和节奏（短句为主，口语化，避免长难句）
- 禁止 AI 套话（"在当今时代"、"综上所述"、"值得注意的是"）
- 随机植入个人化表达（"用了一周后..."、"说实话..."、"真实感受是..."）
- 不同风格对应不同的语气词库

### 3.3 数据模型

```sql
-- Supabase Auth 自动管理 users 表

-- 文案生成记录
generations (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     UUID REFERENCES users(id),
  topic       TEXT NOT NULL,          -- 用户输入的主题
  content_type TEXT NOT NULL,         -- 种草/干货/测评/生活/Vlog
  style       TEXT NOT NULL,          -- 可爱/知性/真实/搞笑/专业
  titles      JSONB NOT NULL,         -- ["标题1", "标题2", "标题3"]
  body        TEXT NOT NULL,          -- 正文
  tags        JSONB,                  -- ["#话题1", "#话题2", ...]
  created_at  TIMESTAMPTZ DEFAULT now()
)

-- 订阅记录
subscriptions (
  id          UUID PRIMARY KEY,
  user_id     UUID REFERENCES users(id) UNIQUE,
  plan        TEXT NOT NULL,          -- 'pro' | 'pro_plus'
  status      TEXT NOT NULL,          -- 'active' | 'expired' | 'cancelled'
  started_at  TIMESTAMPTZ NOT NULL,
  ends_at     TIMESTAMPTZ,
  created_at  TIMESTAMPTZ DEFAULT now()
)

-- 每日额度追踪
usage_logs (
  id          UUID PRIMARY KEY,
  user_id     UUID REFERENCES users(id),
  date        DATE NOT NULL,
  count       INTEGER DEFAULT 1,
  created_at  TIMESTAMPTZ DEFAULT now(),
  UNIQUE(user_id, date)
)
```

### 3.4 API 设计

| 端点 | 方法 | 说明 | 认证 |
|------|------|------|------|
| `/api/generate` | POST | 生成文案 | 否（有限额） |
| `/api/history` | GET | 获取历史记录 | 是 |
| `/api/history/:id` | DELETE | 删除记录 | 是 |
| `/api/usage` | GET | 查询今日剩余次数 | 否 |

### 3.5 安全

- Supabase Row Level Security (RLS) — 用户只能访问自己的数据
- API 限流 — 未登录用户 IP 级别限流
- Prompt 注入防护 — 清洗用户输入，禁止指令注入

### 4.1 定价

|------|------|------|
| 免费 | ¥0 | 3 次/天，5 种基础风格 |

| Pro+ | ¥49.9/月 | Pro 全部 + 专属人设训练 + 新功能内测资格 |

### 4.2 支付方案（分阶段）

**第一阶段 (0~50 付费用户): 手动收款**
- 使用爱发电 (afdian.net) 创建赞助页面
- 用户赞助后手动在 Supabase 中开通会员
- 零手续费、零门槛

**第二阶段 (50~200 付费用户): 半自动**
- 注册个体户（¥300 一次性成本，此时已有月收入覆盖）
- 接入蓝兔支付或 payjs（个人可用的微信支付网关）

**第三阶段 (200+ 付费用户): 正规化**
- 直接接入微信支付商户

### 4.3 收入预测

| 时间节点 | 注册用户 | 付费率 | 付费用户 | 月收入 |

| 第 1 个月 | 500 | 5% | 25 | ~¥500 |

### B. Prompt 模板示例

```
你是一个小红书文案专家。请根据以下信息生成一篇小红书笔记：

【主题】：{用户输入的主题}
【内容类型】：{种草/干货/好物测评/生活记录/Vlog}
【人设风格】：{可爱/知性/真实分享/搞笑/专业}

要求：
- 提供 3 个备选标题（含 emoji）
- 正文 200-500 字，口语化，短句为主
- 推荐 5-8 个话题标签
- 绝对禁止使用以下表达："在当今时代"、"综上所述"、"值得注意的是"、
  "随着...的发展"、"因此"、"此外"、"总而言之"
- 如果风格是"可爱"，多用"姐妹们"、"真的"、"超"、"绝绝子"等词
- 如果风格是"真实分享"，用第一人称，像在跟朋友聊天
- 如果风格是"专业"，保持专业但不生硬，用数据和事实说话
- 加入适当的个人化表达，例如"说实话"、"用了三天之后"、"一开始我也怀疑"等

输出格式（严格JSON）：
{
  "titles": ["标题1", "标题2", "标题3"],
  "body": "正文内容...",
  "tags": ["#标签1", "#标签2", ...]
}
```
